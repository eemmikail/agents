Google AI API Project Documentation

This document provides a comprehensive guide to the Google AI API project that uses Google's Gemini model to generate English adjectives with their definitions and examples.

Project Overview

This project demonstrates how to use Google's Generative AI API (Gemini) to generate structured data in JSON format. Specifically, it requests the model to generate 10 English adjectives along with their definitions and usage examples.

Key features of this project:

  • Integration with Google's Gemini AI model
  • Structured data generation using Pydantic models
  • Environment variable management for API keys
  • JSON schema validation for responses

Project Structure

The project consists of the following files:

  • apiRequest.py - Main Python script that makes the API request
  • .env - Environment file containing the API key
  • Pipfile and Pipfile.lock - Dependency management files
  • document.html - This documentation file

Setup and Installation

Prerequisites

  • Python 3.11 or higher
  • Pipenv for dependency management
  • Google AI API key

Installation Steps

  1. Clone the repository to your local machine
  2. Navigate to the project directory
  3. Install dependencies using Pipenv: 
    pipenv install
  4. Create a .env file in the google_ai_api directory with your API key: 
    GOOGLE_AI_API_KEY="your-api-key-here"
Note: You need to obtain a Google AI API key from the Google AI Studio.

Code Explanation

Pydantic Model

The project uses Pydantic to define a data model for the adjectives: 

class EnglishAdjectives(BaseModel):
    word: str
    definition: str
    example: str

This model ensures that the response from the API contains the expected fields and data types.

Environment Setup

The project uses the python-dotenv library to load environment variables from the .env file: 

load_dotenv()
api_key = os.getenv("GOOGLE_AI_API_KEY")

API Client Initialization

The Google Generative AI client is initialized with the API key:

client = genai.Client(api_key=api_key)

API Request

The request to the Gemini model is made with specific parameters:

response = client.models.generate_content(
    model="gemini-2.0-pro-exp-02-05",
    contents="Generate 10 English adjectives with their definitions and examples",
    config={
        "response_mime_type": "application/json",
        "response_schema": list[EnglishAdjectives]
    }
)

This request:

  • Uses the "gemini-2.0-pro-exp-02-05" model
  • Asks for 10 English adjectives with definitions and examples
  • Specifies that the response should be in JSON format
  • Provides a schema for the response based on the Pydantic model

Running the Project

To run the project, activate the Pipenv shell and execute the Python script: 

pipenv shell
cd google_ai_api
python apiRequest.py

The script will output the first adjective from the response. You can modify the script to display all adjectives if needed.

Example Output

The output will be a structured JSON object containing adjectives, for example: 

{
    "word": "vibrant",
    "definition": "Full of energy, brightness, and life",
    "example": "The market was filled with vibrant colors and sounds."
}

Extending the Project

Here are some ways you could extend this project:

  • Create a web interface to display the generated adjectives
  • Add functionality to save the results to a database
  • Modify the prompt to generate different types of content
  • Implement error handling and retries for API requests
  • Add filtering or search capabilities for the generated content

Troubleshooting

Warning: Never commit your .env file to version control. It contains sensitive API keys.

Common Issues

Issue Solution
API key not found Ensure your .env file is in the correct location and contains the API key.
Module not found errors Make sure you've installed all dependencies with pipenv install.
API rate limiting Google AI API may have rate limits. Implement exponential backoff if making multiple requests.
